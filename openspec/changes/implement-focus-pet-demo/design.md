# 设计：Focus Pet Demo

## 技术方向

- 使用 Flutter 创建可运行 demo。
- 以 iOS-first 为视觉和交互优先级，但保留 Flutter 默认跨端结构。
- 所有状态先放在内存中，确保 demo 可快速运行；后续再接本地持久化。
- soft-focus 通过全屏计时、温和放弃确认、完成奖励和宠物反馈实现。

## 页面结构

1. 引导与宠物选择。
2. 专注计时主页。
3. 宠物成长与解锁区域。
4. 简单统计区域。
5. 分享卡预览区域。

为了让 demo 更直观，首版把这些区域放在一个可滚动主界面中，减少导航复杂度。

## 数据模型

- `FocusSession`：记录专注时长、状态和开始时间。
- `PetProfile`：记录宠物、等级、能量、心情和装饰。
- `RewardLedger`：把完成的专注会话转换为能量和等级。

## 资产策略

使用生成式像素资产，放在 `assets/pixel_pets/`。如果生成式工具不可用，则用 Flutter 自绘像素风组件作为降级方案；但最终 demo 必须有可见的像素宠物视觉。

## 验证策略

- 创建临时 Flutter test 验证奖励逻辑和放弃会话不发能量。
- 运行 `flutter analyze`。
- 运行 `flutter test`，超时时间不低于 30 秒。
- 验证通过后删除临时测试文件。
