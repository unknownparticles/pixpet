# 变更：定义 UI、番茄钟和激励机制

## 为什么

现有 PRD 已经定义 Focus Pet 的产品方向和核心闭环，但 UI 设计、番茄钟行为和激励机制还不够具体。后续继续实现主页、设置页、专注流程、购买食物和购买宠物形象时，需要更明确的设计规格，避免界面只是功能堆叠，也避免奖励系统变成随意数值。

## 变更内容

- 在 PRD 中补充 UI 设计：信息架构、首次进入、主页布局、视觉风格和交互原则。
- 在 PRD 中补充番茄钟设计：时长模式、会话状态、soft-focus 交互和休息设计。
- 在 PRD 中补充激励机制：资源、奖励计算、成长节奏、商店消耗和连续回访规则。
- 在 OpenSpec 中新增可验证需求，约束 UI、计时和奖励实现。

## 不包含

- 不修改 Flutter 代码。
- 不新增硬拦截。
- 不新增付费随机抽卡。
- 不执行 OHOS 检查。
