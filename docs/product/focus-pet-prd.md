# Focus Pet MVP PRD

日期：2026-06-09

## 1. 产品摘要

Focus Pet 是一款原创的学生专注陪伴产品，灵感来自 PixPet、Focus Friend 和 Finch 的公开成功模式。产品通过“专注计时 + 像素宠物成长”帮助学生完成学习会话，让真实努力变成可见成长。

补充 GitHub 同类产品调研后，产品应进一步吸收 PomodoroCat 的即时奖励、Study Buddy 的学习目标、Noto 的连续与统计、WebPet 的轻状态养成、Buddy 的真实任务 XP，以及桌宠产品的可爱陪伴和分享传播。但这些能力必须分阶段进入，MVP 不追求“大而全”。

MVP 已确认使用 Flutter，首发策略为 iOS-first。首版应先验证一个核心闭环，再扩展大型游戏系统：

专注会话 -> soft-focus -> 陪伴进度 -> 领取奖励 -> 收藏/装饰解锁 -> 分享进度 -> 下一次专注。

## 2. 目标用户

### 主要用户

正在准备考试、作业、语言学习、编码练习或证书学习的学生。他们希望用更温和、更有反馈感的方式减少手机分心。

### 次要用户

已经使用番茄钟，但觉得普通效率工具枯燥、压力大、难坚持的年轻职场人。

## 3. 用户问题

- “我知道应该学习，但总会打开短视频或社交 App。”
- “普通计时器太干，进度感不真实。”
- “我想要轻量陪伴，而不是另一个复杂到让我分心的游戏。”
- “我想留下可爱的努力证明，也愿意分享。”

## 4. 产品定位

对于难以开始和完成专注学习的学生，Focus Pet 是一款温和的专注陪伴应用。它把真实学习时间转化为宠物成长和收藏进度。相比沉重的习惯工具或纯计时器，它提供即时情绪反馈，同时保持核心流程简单。

## 5. MVP 目标

- 帮助用户在打开 App 后 3 分钟内完成第一次专注启动。
- 让第一次奖励足够即时、清晰、可理解。
- 通过宠物成长和小收藏制造每日回访理由。
- 生成至少一种可分享成果物，支持自然传播。
- MVP 阶段个人数据本地优先，除非用户后续明确同意云同步。
- 验证“真实专注努力可以养成可爱宠物”的情感动机是否成立。

## 6. MVP 不做什么

- 不做付费随机抽卡。
- 不做完整社交网络。
- 不做宠物战斗系统。
- 不做 NFC 或兑换码经济。
- 不做复杂地图探索。
- 不要求云账号。
- 不包含 OHOS 范围。
- 不做硬拦截。

## 7. 核心用户旅程

1. 用户打开 App，选择一个初始宠物。
2. 用户设置专注时长，默认 25 分钟。
3. 用户进入 soft-focus 模式；MVP 不做硬拦截。
4. 用户开始专注，宠物进入陪伴学习动画。
5. 用户完成会话并获得能量。
6. 能量用于喂养宠物、推进成长或解锁小装饰。
7. 用户看到清晰的进度卡片，包括本次时长、连续天数、宠物心情和下一个解锁目标。
8. 用户可以保存或分享卡片。
9. 用户因为提醒、连续天数、未完成解锁或每日宠物状态再次回来。

## 8. 功能需求

### 8.1 引导

- 用一句话解释产品承诺：“专注学习，获得能量，养成你的专注宠物。”
- 让用户从三个初始宠物中选择一个。
- 只在解释学习提醒后请求通知权限。
- MVP 使用 soft-focus，不请求 App 硬拦截权限。
- 不要求创建账号。

### 8.2 专注计时

- 支持预设时长：15、25、45、60 分钟。
- 支持 5 到 120 分钟的自定义时长。
- 提供暂停和放弃流程。
- 完成会话后发放能量。
- 提前放弃默认不发放能量，但文案应温和，不惩罚用户。

### 8.3 Soft-Focus 模式

- MVP 只做 soft-focus。
- 首版不实现硬拦截，也不接入基于 Screen Time 的 App blocking。
- 提供全屏计时、通知、放弃确认和会话后诚实反馈。
- 产品不得阻断紧急电话或系统关键操作。

### 8.4 宠物成长

- 每只宠物包含名称、心情、等级、能量和成长阶段。
- 第一次升级应在完成一次专注后即可达到。
- 完成专注后宠物必须立刻出现可见变化，至少包含表情、能量、饱腹感、等级或下一个解锁目标中的一种。
- 宠物状态反映近期行为：
  - 今日已专注：开心。
  - 今日未专注：等待。
  - 放弃会话：鼓励，而不是悲伤或羞辱。
- 宠物不得因为用户没有学习而死亡、生病或展示羞辱性反馈。
- MVP 包含三个初始宠物和九个可解锁装饰。
- MVP 视觉资产使用生成式像素图。生成资产必须原创且风格统一。

### 8.5 收藏与奖励

- 能量是 MVP 唯一货币。
- 奖励通过确定性进度解锁，不通过付费随机抽取。
- 收藏内容包含装饰、学习桌物件和简单宠物表情。
- 下一个解锁目标必须始终可见。

### 8.6 统计

- 展示今日专注分钟数。
- 展示本周专注分钟数。
- 展示当前连续天数。
- 展示已完成专注次数。
- 图表要简单，初学用户也能一眼看懂。
- 统计文案应强调用户已经完成的努力，而不是强调失败、缺勤或落后。

### 8.7 分享卡

- 完成专注后生成方形分享卡。
- 分享卡包含宠物图、专注分钟数、连续天数和一句短文案。
- 默认不展示用户输入的私密任务名称。
- 实现阶段保存为本地图片，并在可行时调用系统分享面板。

### 8.8 提醒

- 允许用户设置每日学习提醒。
- 提醒文案应积极、温和，并带有宠物陪伴感。
- 不骚扰用户；MVP 每天一个提醒足够。

### 8.9 UI 设计

MVP 的 UI 目标是让用户第一眼知道“这不是冷冰冰的计时器，而是一只宠物在陪我学习”。视觉应服务专注，不做复杂游戏大厅。

#### 信息架构

- 底部导航包含三项：主页、商店、设置。
- 主页承担最高频任务：查看宠物、开始专注、查看今日进度和分享成果。
- 商店承担消耗能量：购买食物、装饰和宠物形象。
- 设置承担低频配置：提醒、声音、soft-focus 偏好和隐私说明。

#### 首次进入

- 首屏展示三只初始宠物卡片。
- 每张卡片包含大号宠物、名称、性格标签、适合学习场景和选择按钮。
- 用户选中宠物后进入主页，不要求注册或授权。

#### 主页布局

- 顶部展示产品名、今日鼓励语和能量余额。
- 第一屏主体是宠物房间：大号像素宠物、当前心情、饱腹感、等级进度和下一个解锁目标。
- 宠物房间下方是番茄钟控制区：时长选择、开始按钮、暂停/继续、放弃入口。
- 继续向下展示今日分钟、本周分钟、连续天数、完成次数和分享卡预览。

#### 视觉风格

- 风格关键词：像素玩具、柔和、可爱、学习房间、低压力。
- 卡片圆角保持 8px 或更小，避免过度圆润的塑料感。
- 色彩避免单一色系，建议使用奶油底色、蜜桃强调色、青绿成长色、薰衣草休息色和深棕文字。
- 不使用大面积营销式 hero、不使用装饰性渐变球、不用复杂拟物阴影。

#### 交互原则

- 开始专注按钮必须始终醒目，主页不让商店和统计抢走主要注意力。
- 购买、喂食、换装都应给即时反馈，例如宠物表情变化、能量减少和状态条变化。
- 所有失败反馈都使用温和文案，例如“这次先休息一下，下次再陪它完成一段”。

### 8.10 番茄钟设计

MVP 的番茄钟不做硬拦截，重点是降低开始成本，并把完成结果转化为宠物成长。

#### 时长模式

- 默认推荐 25 分钟。
- 预设时长为 15、25、45、60 分钟。
- 自定义时长支持 5 到 120 分钟。
- 第二阶段支持番茄循环：专注 25 分钟、短休息 5 分钟、四轮后长休息 15 分钟。

#### 会话状态

- 未开始：展示宠物等待、推荐时长和开始按钮。
- 进行中：展示倒计时、宠物陪伴动画、暂停按钮和低调的放弃入口。
- 暂停中：停止倒计时，展示继续按钮和温和提醒。
- 已完成：展示奖励结算、宠物反馈、分享卡入口和“再来一段”按钮。
- 已放弃：默认不发放能量，展示鼓励文案，不做惩罚动画。

#### Soft-Focus 交互

- 开始后进入沉浸式计时界面，减少商店、设置和统计干扰。
- 放弃需要二次确认，确认文案强调“可以先休息”，不责备用户。
- 离开 App 后 MVP 不阻止用户操作其他 App；回到 App 时只做诚实反馈。
- 完成后优先展示奖励，不强制用户马上开启下一轮。

#### 休息设计

- MVP 可以先用完成后的休息建议文案，不强制进入休息倒计时。
- 第二阶段再加入短休息和长休息计时。
- 休息期不发放专注能量，但可以展示宠物吃东西、伸懒腰或整理书桌的轻反馈。

### 8.11 激励机制

激励机制必须让用户感觉“奖励是我靠真实努力挣来的”，而不是被登录任务或付费随机牵着走。

#### 资源

- 能量：完成专注获得，是购买食物、装饰和宠物形象的主要资源。
- 饱腹感：通过喂食提升，影响宠物表情和短期心情，不作为惩罚工具。
- 等级：由累计能量推动，代表长期成长。
- 连续天数：每天完成至少一次有效专注后增加。

#### 奖励计算

- 15 分钟专注奖励 8 能量。
- 25 分钟专注奖励 12 能量。
- 45 分钟专注奖励 22 能量。
- 60 分钟专注奖励 30 能量。
- 自定义时长按每 5 分钟 2 能量计算，单次最高 40 能量。
- 第一次完成专注额外奖励 8 能量，确保用户立刻能喂食或接近第一次购买。

#### 成长节奏

- 第一次升级应在 1 到 2 次专注内发生。
- 前三个装饰或食物应在首日可通过 1 到 3 次专注获得。
- 每一级展示下一级所需能量和奖励预览。
- 解锁优先顺序建议为：食物 -> 桌面装饰 -> 表情 -> 宠物形象。

#### 商店与消耗

- 食物价格低，承担高频即时反馈。
- 装饰价格中等，承担房间变化和分享价值。
- 宠物形象价格较高，承担长期目标。
- MVP 不做付费随机抽卡，不售卖影响学习公平性的资源。

#### 连续与回访

- 连续天数只做正向反馈，不做断签惩罚。
- 断签后展示“今天重新开始也很好”，不展示失败天数。
- 每日提醒应围绕宠物陪伴和具体学习时段，而不是制造焦虑。

## 9. 商业化

MVP 应免费或内部测试可用，不依赖付费。后续商业化可以包括：

- 装饰包。
- 高级宠物主题。
- 终身解锁。
- 可选订阅，用于扩展收藏、云同步和高级统计。

考虑学生信任和应用商店风险，付费随机奖励不进入 MVP。

## 10. 指标

- 引导完成率。
- 首次专注开始率。
- 首次专注完成率。
- 奖励领取率。
- D2 和 D7 留存。
- 活跃用户平均完成专注分钟数。
- 分享卡导出率。
- 提醒开启率。

## 11. 发布范围

### MVP

- 本地优先 App。
- 三个初始宠物。
- 计时器和会话记录。
- 能量奖励闭环。
- 基础宠物成长。
- 简单统计。
- 每日提醒。
- 分享卡。
- soft-focus 模式。
- 用于初始宠物、装饰和分享卡视觉的生成式像素资产。

### 第二阶段

- 步数兑换能量。
- 更多宠物和装饰。
- 周报回顾。
- 每日目标和周计划。
- 学习场景标签，例如考试、语言、作业、编码练习。
- 解锁揭晓动画和更多分享卡模板。
- 云同步。
- 好友邀请或双人学习房间。

### 第三阶段

- 轻量地图探索。
- 季节活动。
- 小组学习房间。
- 谨慎设计的排行榜。
- 本地记忆或 AI 宠物对话。
- 桌面端或小组件形态的轻陪伴。

## 12. GitHub 同类产品融合策略

| 来源类型 | 核心洞察 | 进入阶段 |
| --- | --- | --- |
| PomodoroCat | 专注奖励要即时、可爱、可看见 | MVP |
| Study Buddy | 学习目标和宠物状态可以绑定 | 第二阶段 |
| Noto | 连续天数、等级和积极反馈能降低枯燥感 | MVP 简化版，第二阶段增强 |
| WebPet | 饱腹感、心情和稀有形象能形成回访理由 | MVP 简化版，第二阶段增强 |
| Buddy | 真实任务带来的 XP 比登录奖励更可信 | MVP |
| 桌宠产品 | 可爱常驻形象和截图传播有天然吸引力 | MVP 做房间和分享卡，桌面形态延后 |
| AI 虚拟宠物 | 个性化陪伴有潜力，但成本和复杂度较高 | 第三阶段 |

融合原则：

- 只借鉴公开产品模式，不复制任何同类产品的代码、素材、角色或文案。
- 所有成长资源优先来自真实专注、步数或学习目标，不把每日登录作为主要经济来源。
- MVP 聚焦即时奖励和温和陪伴；学习目标、稀有度、AI 伴侣、桌宠形态按留存数据逐步加入。
- 宠物反馈必须帮助用户回来学习，而不是制造压力、羞耻或付费焦虑。

## 13. 已确认实现决策

- 客户端技术栈：Flutter。
- 首发平台：iOS-first。
- 专注模式：先做 soft-focus；MVP 不做硬拦截。
- 视觉生产路径：生成式像素资产。
