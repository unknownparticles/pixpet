## ADDED Requirements

### Requirement: 即时可见奖励

MVP SHALL 在用户完成专注后立刻展示宠物或成长状态变化，让用户感知本次努力已经产生结果。

#### Scenario: 用户完成第一次专注

- **WHEN** 用户完成一次专注会话
- **THEN** App 立即展示能量、等级、饱腹感、宠物表情或下一个解锁目标中的至少一种变化
- **AND** 用户不需要进入深层页面才能理解奖励结果。

### Requirement: 非惩罚性宠物状态

MVP SHALL 使用温和宠物状态鼓励用户继续学习，不用死亡、生病、羞辱或强压力反馈惩罚用户缺勤。

#### Scenario: 用户今天还没有专注

- **WHEN** 用户打开 App
- **THEN** 宠物可以展示等待、撒娇或鼓励状态
- **AND** 不展示死亡、生病、羞辱或责备用户的文案。

### Requirement: 分享传播优先于复杂社交

MVP SHALL 优先提供分享卡和宠物房间截图价值，而不是完整社交网络。

#### Scenario: 用户完成专注后想分享

- **WHEN** 用户查看完成结果
- **THEN** App 提供包含宠物、专注分钟数和连续天数的分享卡
- **AND** MVP 不要求用户加入好友关系或社交网络才能分享成果。
