# 项目概览

这是一个受 PixPet 公开产品模式启发的“专注陪伴宠物”产品规划与实现仓库。

## 当前状态

- 仓库类型：从空仓库开始的产品与工程规划仓库。
- 实现技术栈：已确认使用 Flutter。
- 首发平台：已确认 iOS-first。
- 首版专注策略：先做 soft-focus，不做硬拦截。
- 像素资产路径：首版使用生成式像素资产。
- 产品方向：面向学生的番茄钟专注与像素宠物养成。
- OpenSpec 状态：先通过 proposal 定义产品范围，再进入实现变更。

## 产品原则

- 做原创产品，只借鉴公开市场模式，不复制 PixPet 品牌、素材、文案或私有实现。
- 未验证的市场判断必须标为假设，不能写成事实。
- MVP 足够小：先验证“专注会话 -> 奖励 -> 宠物成长 -> 分享/回访”闭环，再做复杂社交和游戏系统。
- 需求、任务和后续代码都要让初学者也能读懂。

## 仓库约束

- 修改 Flutter 项目时不执行 OHOS 相关检查。
- 不主动格式化代码。
- 后续新增代码要写必要但不过量的注释，解释用途、输入输出和非直观逻辑。
- 临时测试类只用于验证；验证完成后按仓库规则删除。

## 已确认实现方向

- 客户端：Flutter。
- 平台策略：iOS-first，同时保留 Flutter 跨端可能性。
- 本地存储：建议先用简单 repository 边界封装，具体 SQLite/Isar/轻量本地文件在实现 proposal 中再定。
- 后端：MVP 不需要后端；云同步等留到留存验证后。
- 专注拦截：MVP 只做 soft-focus，包括全屏计时、提醒、温和放弃流程和完成后反馈，不接入硬拦截。
- 像素资产：MVP 使用生成式像素资产，保证原创并保留后续替换为手绘资产的空间。
- 数据指标：记录激活、首次开始专注、完成专注、领奖、分享卡导出、D2/D7 留存等隐私友好事件。
