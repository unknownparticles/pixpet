# 设计：主页、设置和商店养成 Demo

## 结构

Demo 使用底部导航分为三个页签：

1. 主页：宠物、专注、统计、分享卡。
2. 商店：食物购买、喂食、宠物形象购买和切换。
3. 设置：提醒、柔和提示、声音反馈、数据重置。

## 数据模型

- `PetProfile` 增加饱腹感、已拥有宠物形象、已购买食物数量。
- `ShopCatalog` 定义食物和宠物形象商品。
- `ShopService` 处理购买、喂食和切换形象，保证能量不足时不扣费。

## 交互原则

- 能量是 demo 唯一货币，来自完成专注。
- 食物用于喂食，提高饱腹感和宠物心情。
- 宠物形象购买后可切换，不重复扣费。
- 设置页只影响本地 demo 状态，不涉及系统权限或真实通知。

## 验证

- 临时测试验证：能量足够时购买成功、能量不足时购买失败、喂食消耗库存并提升饱腹感、已购买形象可切换。
- 验证后删除临时测试文件。
- 运行 `flutter analyze`。
