# 设计：同类产品洞察到 PRD 的融合方式

## 分层原则

同类产品不是功能清单，而是帮助 Focus Pet 排序。融合时按三个层级处理：

1. MVP 必须吸收的底层机制：即时奖励、真实专注换成长、温和宠物反馈和可分享成果。
2. 第二阶段增强的习惯系统：每日目标、周计划、学习标签、周报、稀有形象和更多分享模板。
3. 第三阶段探索的陪伴形态：AI 宠物对话、本地记忆、桌面端或小组件形态。

## 产品取舍

PomodoroCat 证明“iOS 番茄钟 + 可爱宠物奖励”可以很轻，所以 MVP 应优先让第一次完成专注后立刻有变化。Study Buddy 和 Noto 的目标、连续和统计适合提升长期留存，但如果一开始做得太重，会干扰首次专注闭环。WebPet 的状态衰减能制造回访理由，但 Focus Pet 面向学习场景，不能用死亡、生病或羞辱制造压力。

Buddy 和桌宠产品说明“真实任务反馈 + 可爱角色”有传播潜力，因此 Focus Pet 的分享卡和宠物房间应成为首版传播载体。AI Tamago 类产品代表远期机会，但在留存验证前不把大模型对话作为核心卖点。

## 文档落点

- 市场调研记录同类产品观察和融合原则。
- PRD 记录阶段边界和具体功能要求。
- OpenSpec 记录可验证需求，保证后续实现能按阶段落地。
