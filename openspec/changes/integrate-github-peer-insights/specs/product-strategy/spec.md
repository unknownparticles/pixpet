## ADDED Requirements

### Requirement: GitHub 同类产品洞察分层

产品策略 SHALL 将 GitHub 同类产品洞察分为 MVP、第二阶段和第三阶段，避免把所有竞品能力一次性塞进首版。

#### Scenario: 团队规划功能路线

- **WHEN** 团队阅读 PRD 或 OpenSpec
- **THEN** 团队可以看到 PomodoroCat、Noto、WebPet 和 Buddy 的核心机制如何影响 MVP
- **AND** 可以看到 Study Buddy 式学习目标属于第二阶段
- **AND** 可以看到 AI 宠物和桌宠形态属于第三阶段或后续探索。

### Requirement: 公开借鉴边界

产品策略 SHALL 只借鉴公开同类产品的模式和经验，不复制同类产品的代码、角色、视觉素材、品牌或专有文案。

#### Scenario: 团队从 GitHub 项目提炼功能

- **WHEN** 团队将 GitHub 同类产品写入调研或 PRD
- **THEN** 文档说明这些项目只作为公开参考
- **AND** 文档明确 Focus Pet 仍然是原创产品。

### Requirement: 真实努力作为核心经济来源

产品策略 SHALL 将真实专注、步数或学习目标完成作为主要成长来源，不把每日登录奖励作为核心经济来源。

#### Scenario: 团队设计奖励系统

- **WHEN** 团队设计能量、经验或宠物成长
- **THEN** 主要奖励来源来自专注完成、后续步数兑换或学习目标完成
- **AND** 每日登录不得成为驱动宠物成长的主要资源入口。
