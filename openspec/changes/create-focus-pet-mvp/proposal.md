# 变更：建立 Focus Pet MVP 产品基础

## 为什么

当前仓库是空仓库，需要先建立产品方向、需求范围和工程推进口径。基于 PixPet、Focus Friend、Finch 等公开资料，可以看到“专注/自我管理 + 可爱陪伴体 + 可视化成长”具备传播和留存潜力。我们要做的是原创的学生专注陪伴产品，而不是复制 PixPet。

## 变更内容

- 增加 PixPet 相关市场调研与爆款原因分析。
- 增加 Focus Pet MVP PRD。
- 明确已确认实现方向：Flutter、iOS-first、MVP 先做 soft-focus、不做硬拦截、像素资产使用生成式资产。
- 定义 MVP 需求：引导、专注计时、soft-focus、宠物成长、确定性奖励、统计、提醒和分享卡。
- 通过 OpenSpec tasks 建立可追踪的推进任务。

## 影响范围

- 为仓库建立第一版产品规格。
- 增加产品策略和 MVP 行为两个 OpenSpec 能力增量。
- 为后续 `implement-focus-pet-mvp` 实现变更提供明确输入。

## 不包含

- 本 proposal 不写应用代码。
- MVP 不做付费随机抽卡、完整社交网络、宠物战斗、NFC、复杂地图、云同步或 OHOS 工作。
- 不使用 PixPet 品牌、素材、文案或私有实现。
